## 短期（第 3-4 月）— 模型与工程深度优化

### 1. int8 量化
- 当前: fp16 量化，14.5 MB
- 目标: int8 量化，压到 ~7 MB
- 方法: PyTorch `torch.quantization` 静态量化
- 收益: 加载速度再翻倍，体积再减半
- 难度: ⭐⭐

### 2. ONNX 导出 + ONNX Runtime
- 当前: 纯 PyTorch 推理
- 目标: V1 模型导出为 ONNX，用 ONNX Runtime 推理
- 收益: 推理速度 2-3x
- 注意: ESM-2 有复杂 op 可能不兼容，先做 V1
- 难度: ⭐⭐⭐

### 3. 换更大的 ESM-2 变体 + 重训练
- 当前: esm2_t6_8M（8M 参数）
- 可选: esm2_t12_35M（35M）、esm2_t30_150M（150M）
- 收益: 预期 Q3 可提升 1-2%
- 代价: 训练时间从 2 小时变 8-24 小时
- 难度: ⭐⭐

### 4. 预测结果缓存
- 当前: 每次预测都重新跑
- 目标: 用序列哈希作为 key，相同序列直接返回缓存结果
- 收益: 重复查询秒出
- 方法: 简单 dict 缓存 + LRU 淘汰
- 难度: ⭐

### 5. 响应压缩 + 图片清理
- 当前: statc/ 目录的旧图片不清理
- 目标: 服务器定时清理 1 小时前的图片
- 收益: 避免磁盘占用无限增长
- 难度: ⭐

## 中期（第 5-7 月）— 论文/比赛准备

### 6. 构建测试数据集
- 从 PDB 下载 500-1000 条蛋白质结构
- 提取序列 + DSSP 标注的二级结构
- 划分训练/验证/测试集
- 格式: FASTA 序列 + FASTA 结构
- 难度: ⭐⭐

### 7. Ablation Study（消融实验）
对比以下配置在测试集上的 Q3 准确率:
| 配置 | V1 | ESM-2 | 平滑规则 | 预期 Q3 |
|------|----|-------|---------|---------|
| 完整集成 | ✅ | ✅ | ✅ | ~89.8% |
| 仅 V1 | ✅ | ❌ | ✅ | ~85.5% |
| 仅 ESM-2 | ❌ | ✅ | ✅ | ~87.3% |
| 无平滑 | ✅ | ✅ | ❌ | ~89.0% |
- 收益: 论文/报告的核心数据
- 难度: ⭐⭐

### 8. Benchmark 对比
跟以下工具在同一测试集上对比:
- NetSurfP-2.0（主流，有网页版）
- PSIPRED（经典工具）
- SPOT-1D（深度学习）
- 指标: Q3 Accuracy、SOV、每类 F1
- 收益: 论文最有说服力的部分
- 难度: ⭐⭐⭐

### 9. 置信度校准
- 当前: confidence 数值不可靠，同样预测可能给不同置信度
- 方法: Temperature Scaling / Platt Scaling
- 收益: confidence 更有参考意义
- 难度: ⭐⭐

## 中长期（第 8-10 月）— 扩展新功能

### 10. 溶剂可及性预测（ASA）
- 预测每个残基在蛋白质表面还是内部
- 跟二级结构类似，可以用 ESM-2 微调做
- 需要 PDB 数据 + DSSP 提取 ASA 标签
- 难度: ⭐⭐⭐

### 11. 无序蛋白预测
- 预测蛋白质哪些区域是无序的（ intrinsically disordered）
- 跟二级结构互补
- 可以直接用 ESM-2 嵌入 + 分类器
- 难度: ⭐⭐

### 12. 3D 结构查看器
- 集成 Mol* 或 NGL Viewer
- 把预测的二级结构映射到 3D 模型上
- 上传 PDB 文件查看
- 收益: 视觉效果拉满，答辩/展示加分
- 难度: ⭐⭐⭐

### 13. Web 服务增强
- 加 Nginx 反向代理
- HTTPS 支持
- 多进程/异步处理
- WebSocket 实时进度
- 难度: ⭐⭐⭐
