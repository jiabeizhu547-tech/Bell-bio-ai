## 二、数据集准备（第 1-2 月）

这是论文最基础也最花时间的一步。没有标准测试集，论文的数据不可信。

### 2.1 训练集
| 来源 | 数据量 | 用途 | 获取方式 |
|------|--------|------|----------|
| PDB (蛋白质数据库) | 5000-10000 条 | 训练 + 验证 | 从 PDB 下载结构，DSSP 提取二级结构 |
| CB513 数据集 | 513 条 | 标准测试集 | 学术界广泛使用的 benchmark |
| CASP 测试集 | ~100 条 | 额外测试 | CASP 比赛公开数据 |

### 2.2 测试集
| 数据集 | 来源 | 用途 |
|--------|------|------|
| CB513 | 经典公开数据集 | 跟所有工具对比的标准 |
| TEST2016 | NetSurfP 提供的测试集 | 跟 NetSurfP 对比 |
| CASP15 | 最新 CASP 数据 | 展示泛化能力 |

### 2.3 突变效应评估集
| 数据集 | 来源 | 用途 |
|--------|------|------|
| ClinVar | NCBI 临床变异数据库 | 评估致病性预测 AUC |
| ProteinGym | 综合突变数据集 | Deep Mutational Scanning 数据 |

### 2.4 实现步骤

```bash
# 第 1 步: 下载 PDB 结构文件
# 可以用 PDB 的批量下载工具或 rsync 镜像

# 第 2 步: 用 DSSP 提取二级结构
# dssp 每个 PDB 文件 → 氨基酸 + H/E/C 标签

# 第 3 步: 清洗、去冗余（序列相似度 < 25%）
# 用 CD-HIT 去冗余

# 第 4 步: 格式化为 FASTA 对
# >protein_name
# MKVLILACLVALALACTVQA...
# >structure
# CHHHHHHHHHHHHHHHCCCCC...
```

**预计耗时：2-4 周（主要花在下载和清洗上）**

| 实验编号 | 配置 | 变量 |

| E6 | V1 + ESM-2 + 平滑 + int8 量化 | 量化影响 |

**指标：** Q3 Accuracy、SOV（Segment Overlap）、每类 Precision/Recall/F1

- 折线图：不同序列长度下的准确率

## 六、每个实验的具体操作步骤

### Ablation Study 操作（以 E2 仅 V1 为例）

```
1. 修改 inference/inference.py 中的集成权重
   ENSEMBLE_W1 = 1.0, ENSEMBLE_W2 = 0.0

2. 在测试集上运行推理
   for seq, true_ss in test_set:
       pred = predict_secondary_structure(seq)
       计算 Q3, F1 per class

3. 记录结果到表格

4. 恢复权重
   ENSEMBLE_W1 = 0.4, ENSEMBLE_W2 = 0.6
```

### Benchmark 对比操作（以 NetSurfP 为例）

```
1. 注册 NetSurfP 网站账号（可能需要教育邮箱）
2. 获取 API token 或下载命令行工具
3. 将测试集格式化为 NetSurfP 要求的输入格式
4. 运行 NetSurfP 预测
5. 解析 NetSurfP 输出，提取二级结构预测
6. 跟真实标签对比，计算 Q3 / SOV / F1
7. 跟本项目的预测结果对比
```
